# ATAC-seq Upstream Pipeline

Fork of [`../cutrun/`](../cutrun/README.md) for ATAC-seq libraries.
Same seven-step layout, config-driven, with **MACS3 + HOMER peak calling** kept side-by-side.

## What's different from CUT&RUN

| Step | CUT&RUN | ATAC |
|------|---------|------|
| 1 trim | Trim Galore default (TruSeq) | `--nextera` (Tn5 ME adapter `CTGTCTCTTATA`) |
| 2 align | `bowtie2 --very-sensitive` | adds `-X 2000 --no-mixed --no-discordant` |
| 2 post-align | none | `samtools fixmate + markdup`, MAPQ ≥ 30, drop chrM, **Tn5 shift via `alignmentSieve --ATACshift`** |
| 3 tags | `*_sorted.bam` | `*_sorted.shifted.bam` |
| 4.1 MACS3 | `--format BAMPE` (PE) or BAM (SE), `--broad` for histone | **forced** `--nomodel --shift -75 --extsize 150 -f BAM --keep-dup all` (treats every Tn5 cut site individually) |
| 4.2 HOMER | `-style factor`/`histone`, optional `-i control` | `-style dnase` always, no control |
| 5 QC | `run_spp.R` for ChIP cross-correlation | dropped; **adds bamPEFragmentSize and TSS enrichment** |

## Workflow

```
Raw FASTQs (data/)
   → 1_trim_qc       cleandata/   (Trim Galore --nextera + FastQC)
   → 2_bowtie2       align/bam/   (_marked.bam → _sorted.bam → _sorted.shifted.bam)
                     align/track/ (.bw from shifted BAM)
   → 3_homer_tags    align/tags/  (from shifted BAM)
   → 4.1_peak_macs3  peaks/macs3/ (narrowPeak only)
   → 4.2_peak_homer  peaks/homer/ (findPeaks -style dnase)
   → 5_qc            multiqc/, bamqc/ (preseq, fragment size, TSS enrichment, fingerprint, PCA)
```

## BAM file lineage (per sample)

```
<sample>_marked.bam            # post-fixmate+markdup (dups marked, NOT removed)
                                 used by preseq for library complexity
<sample>_sorted.bam            # filtered: MAPQ ≥ MAPQ_MIN, drop chrM, drop dup/QCfail/etc
                                 used by FRiP, fingerprint, multiBamSummary
<sample>_sorted.shifted.bam    # Tn5 +4/-5 cut-site shifted (alignmentSieve --ATACshift)
                                 used by tag dirs, peak callers, bigWig
```

`_marked.bam` is large; set `DELETE_MARKED_AFTER_BOWTIE2=1` to drop it (preseq won't run on rerun).

## Setup

1. **Copy** this folder into your project as `script/`.
2. **FASTQs** under `${BASE}/data/` (see [`../tools/prep/`](../tools/prep/README.md) for IGM/ENA/lane-merge helpers).
3. **Edit `0_config.sh`**:
   - `BASE` → project root (e.g. `${HOME}/work/seq/ATACseq/my_project`)
   - `GENOME` / `GENOME_INDEX` → bowtie2 index
   - `BLACKLIST` → blacklist BED
   - `TSS_BED` → auto-generated by `5_qc.sh` from `~/work/ref/annot/${GENOME}/${GENOME}_annot_genome.bed` if missing
   - Step toggles `RUN_*` — override per run
4. **Create `peakcall_groups.tsv`** at project root. Format unchanged for parser
   compatibility; for ATAC, controls are always `-` and the type column is informational:
   ```
   sampleA_sorted.shifted.bam    -    sampleA    ATAC
   sampleB_sorted.shifted.bam    -    sampleB    ATAC
   ```

## Usage

```bash
# Full pipeline
./run_all.sh

# Resume from peak calling
RUN_TRIM=0 RUN_BOWTIE2=0 RUN_HOMER_TAGS=0 RUN_PEAK_MACS3=1 RUN_PEAK_HOMER=1 ./run_all.sh

# Single-end
SE=1 ./run_all.sh

# QC only (re-run after peaks change)
RUN_TRIM=0 RUN_BOWTIE2=0 RUN_HOMER_TAGS=0 RUN_PEAK_MACS3=0 RUN_PEAK_HOMER=0 RUN_QC=1 ./run_all.sh
```

## Parameter notes

- **Tn5 shift (`+4`/`-5`)**: required because Tn5 binds as a dimer and inserts adapters with a 9 bp stagger; uncorrected reads systematically misplace cut sites. `alignmentSieve --ATACshift` ([deepTools docs](https://deeptools.readthedocs.io/en/develop/content/tools/alignmentSieve.html)) handles both PE and SE.
- **MACS3 `--shift -75 --extsize 150`**: standard ENCODE ATAC narrow-peak setting; centers a 150 bp window on each cut site. Forced for both PE and SE per project decision (vs `-f BAMPE` which uses fragment ends).
- **`--keep-dup all`**: dedup happened upstream via `samtools markdup`, so MACS3 should not dedup again.
- **MAPQ ≥ 30**: ENCODE ATAC standard; removes multi-mappers.
- **chrM filter**: ATAC libraries are typically 20–80% mitochondrial; removed *after* dup marking so the marked BAM still reflects the full library complexity.
- **HOMER `-style dnase`**: matched DNase-hypersensitivity peak shape; appropriate for ATAC point-source peaks ([HOMER findPeaks docs](http://homer.ucsd.edu/homer/ngs/peaks.html)).
- **TSS enrichment**: a positive ATAC dataset shows clear enrichment within ±500 bp of annotated TSSs; a flat profile suggests poor signal-to-noise.

## Requirements

On `PATH` via `CONDA_BIO_ENV`:
trim_galore, cutadapt, fastqc, bowtie2, samtools (≥1.10 for `markdup`), deeptools
(`alignmentSieve`, `bamCoverage`, `bamPEFragmentSize`, `computeMatrix`, `plotProfile`,
`plotFingerprint`, `plotPCA`, `plotCorrelation`, `multiBamSummary`),
homer (`makeTagDirectory`, `findPeaks`, `annotatePeaks.pl`), bedtools,
macs3, preseq, multiqc.
